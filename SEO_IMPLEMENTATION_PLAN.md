## Technical SEO Implementation Steps

1. Add robust global metadata in app layout
- File: app/layout.tsx
- Add:
  - metadataBase
  - alternates.canonical
  - openGraph
  - twitter
  - robots (index/follow)
  - category/keywords arrays based on above clusters
- Keep existing UI content unchanged.

2. Add route-level metadata using App Router
- Files:
  - app/page.tsx
  - app/explorer/page.tsx
  - app/predictor/page.tsx
  - app/institutes_and_programs/page.tsx
- Add export const metadata (or generateMetadata).
- Use unique, intent-matched title and description per route.

3. Add sitemap and robots files
- Create:
  - app/sitemap.ts
  - app/robots.ts
- Include all public routes and proper lastModified/frequency hints.

4. Add structured data (JSON-LD)
- In layout and key pages:
  - Organization
  - WebSite
  - BreadcrumbList (where applicable)
  - SoftwareApplication (predictor)
  - ItemList (institutes/programs)
- Inject via script tags in server components.

5. Improve technical discoverability
- Verify clean canonical URLs for each page.
- Ensure no accidental noindex in prod.
- Validate static export URLs in next.config.ts deployment context.

6. Image/asset SEO checks
- Ensure logo/image alt text remains meaningful (already mostly present).
- Add social preview image metadata if missing.

7. Internal linking and anchor optimization (without text rewrites)
- Keep existing nav/footer links.
- If needed, adjust aria-label/title attributes to include keyword intent naturally.
- Do not alter visible content sentences.
